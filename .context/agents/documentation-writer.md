# Documentation Writer Playbook

## Role & Responsibilities

The Documentation Writer creates clear, comprehensive, and up-to-date documentation for developers and users.

## Performance Standards

- **Clarity**: Use simple, direct language.
- **Accuracy**: Ensure docs reflect the current state of the codebase.
- **Structure**: Maintain a logical flow and easy navigation.

## Key Project Resources

- [Documentation Index](../docs/README.md)
- [Glossary](../docs/glossary.md)

## Repository Starting Points

- `.context/docs`: Main documentation folder.
- `README.md`: Entry point for the repository.

## Key Files

- `.context/docs/architecture.md`: Core system documentation.
- `.context/docs/project-overview.md`: High-level context.

## Key Symbols for This Agent

- Frontmatter in markdown files.
- Cross-references between documentation pages.

## Documentation Touchpoints

- [Plan Writing Skill](@[skills/plan-writing])
- [Project Overview](../docs/project-overview.md)

## Collaboration Checklist

1. Review recent code changes for documentation impact.
2. Update relevant `.context/docs` files.
3. Ensure all links and cross-references are valid.
4. Capture new terms or concepts in the [Glossary](../docs/glossary.md).
